What is a graph database?

Database that uses a graph structure to store items and their relationships to other items through the use of nodes and edges.


Why are they useful?

  • Good for handling extremely large and complex data
    • No fixed schema:
      • No redundant data
      • Relationships and nodes can be conveniently altered without shifting the design of your database.

    Every relationship and node contains important information whereas in a language like SQL which is relational, it would be difficult to create links between different tables. This allows you to create an extremely complex dataset. For example, a dataset of actors and all the movies they have and haven't acted in would be possible and intuitive using graph databases.

Implementation

Neo4j is a graph database platform. It uses a query language called Cypher which closely resembles SQL, making the syntax intuitive and easy to follow

Syntax

Creating nodes can be done using the CREATE keyword, allowing you to specify attributes of the node. Additionally, you can save the node as a temporary variable which allows us to return it, displaying it on the page.

//Creates two nodes. One for Bob, and one for Stuy
CREATE (p:Person {name: 'Bob', age: 17})
CREATE (s:School {name: 'Stuy'})
RETURN p, s

To create edges, you first need to select the two nodes which you want to link using the MATCH keyword. We can then create the edge using the selected nodes like so.

//Creates link "Bob has studied at Stuy since 2015"
MATCH (bob:Person {name:'Bob'})
MATCH (s:School {name:'Stuy'})
CREATE (bob)-[:STUDIES_AT {Start: 2015}]->(s)
RETURN bob, s
